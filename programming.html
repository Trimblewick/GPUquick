Linux MDADM RAID

Partition all harddrives with linux ee

mdadm --create /dev/md127 --num-raid-devices 4 /dev/sda1 /dev/sdb1 /dev/sdc1 /dev /sdd1

Electronics are complicated. Electrical current is not read as 1 or 0, it is parsed into 1 or 0 from some threshold on the measured current. From my understanding of harddisks, the stored values in a magnetic segment on the disk could be interpreted as a value between zero and one. Is the value bellow or higher than 0.5 is what determens if the bit was 1 or 0. A read error occurs when the value 0.49 is misread as 0.51, which parses the bit to 1 when it should have been 0. Some batches are more reliable than others. Some mechanical harddrives never fails, and some do after just a year.

The read error becomes "unrecoverable" when reconstructing the redundancy of the raid array onto a spare disk. The recrated data is simply corrupted when one of the bits are misread.
